# 💫 About Me

I build ML systems, agentic workflows, backend services, and self-hosted AI tooling that are meant to survive real workloads, real state, and real complexity.

I started from Data Science and model experimentation.  
Now the vector is sharper:

- ML Engineering
- Agentic Systems
- LLM Orchestration
- Backend Architecture
- Linux-first Engineering
- Modern C++
- CUDA and GPU-aware thinking
- Self-hosted AI Infrastructure

I like systems where the model is not the whole product, but one layer inside a larger machine:  
tools, memory, routing, queues, APIs, storage, evaluation, serving, orchestration, and runtime control.

---

# ⚡ What I’m Focused On

### Agentic Systems
Building tool-using AI systems with controllable execution, routing, memory, graph-based workflows, and multi-agent coordination.

### LLM Applications
RAG, OCR + LLM pipelines, context assembly, retrieval quality, evaluation, prompt control, and production-minded LLM integration.

### Backend Systems
FastAPI services, async pipelines, task queues, state handling, structured APIs, and systems where reliability matters more than hype.

### AI Infrastructure
Self-hosted inference workflows, runtime management, backend orchestration, Linux-based environments, and operational control over AI stacks.

### Systems Engineering Direction
Modern C++, CLI tooling, filesystem-level work, architecture boundaries, performance culture, and the long road toward lower-level engineering.

---

# 🧠 Current Vector

Right now I’m moving deeper into:

- **LangChain / LangGraph** for agentic orchestration and execution graphs
- **multi-agent systems** with tools, routing, memory, and supervisor patterns
- **RAG systems** with stronger retrieval and cleaner context assembly
- **OCR + LLM pipelines** for document-heavy workloads
- **FastAPI + Celery + Redis + RabbitMQ + PostgreSQL** as a serious backend spine
- **self-hosted LLM serving** and runtime management across multiple backends
- **modern C++** for system thinking, stronger abstractions, and performance mindset
- **CUDA and GPU-aware engineering** as the deeper layer under AI runtime systems
- **platform thinking**: services, queues, state, observability, and control planes

# 🔥 Engineering Code

I prefer systems that can be inspected, decomposed, extended, and pushed harder without instantly turning into sludge.

What matters to me:
- explicit flow over hand-wavy cleverness
- sharp abstractions over fake magic
- architecture over script piles
- reproducibility over vibes
- control over chaos
- strong boundaries between layers
- systems that work under pressure, not just in screenshots

The point is not to make something that looks smart.
